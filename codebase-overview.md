# Codebase Overview: Blog Genny

## 1. Project Function and Scope

**Blog Genny** is a full-stack web application designed for generating AI-powered blog posts. It allows users to input a topic and receive a complete, SEO-friendly article written by Google's Gemini AI. The application provides a clean, modern interface for reading, managing, and searching for blog content.

The project's scope includes:
- **AI Content Generation**: Core functionality for creating articles from user prompts.
- **Content Management**: Storing and serving blog posts, which appear to be saved as MDX files.
- **User Authentication**: A complete authentication system with sign-in, sign-up, and user-specific dashboards.
- **Premium Features**: An "upgrade" path, suggesting a monetization strategy with premium access for certain features.
- **Administration**: A dedicated admin panel for managing users, articles, and site settings.
- **Search**: Functionality for searching through existing blog posts.

## 2. Key Technologies and Dependencies

- **Framework**: [Next.js](https://nextjs.org/) (v15) with the App Router.
- **Language**: [TypeScript](https://www.typescriptlang.org/).
- **Database & ORM**: [PostgreSQL](https://www.postgresql.org/) with [Drizzle ORM](https://orm.drizzle.team/) for database access and schema management.
- **Authentication**: [NextAuth.js](https://next-auth.js.org/) for handling user sign-in, sign-up, and sessions.
- **AI**: [Google Gemini AI](https://ai.google.dev/) (`@google/generative-ai`) for the core blog generation feature.
- **UI/Styling**:
    - [Tailwind CSS](https://tailwindcss.com/) for utility-first styling.
    - [Shadcn/UI](https://ui.shadcn.com/) for the component library, built on Radix UI.
    - [MDX](https://mdxjs.com/) (`@mdx-js/react`, `next-mdx-remote`) for rendering rich content in blog posts.
- **State Management**: [TanStack Query](https://tanstack.com/query/latest) (`@tanstack/react-query`) for managing server state and caching data.
- **Tooling**:
    - [ESLint](https://eslint.org/) for code linting.
    - [tsx](https://github.com/esbuild-kit/tsx) for running TypeScript scripts.

## 3. Directory Structure and Key Files

- **`app/`**: The core of the Next.js application, following the App Router structure.
    - **`app/actions/`**: Contains server actions, notably `generateBlog.ts` for the AI generation logic.
    - **`app/api/`**: API routes for various functionalities (admin, articles, auth).
    - **`app/admin/`**: The admin dashboard UI and related sub-pages.
    - **`app/blog/[slug]/`**: Dynamic routes for displaying individual blog posts.
    - **`app/dashboard/`**: The user-specific dashboard for managing articles and settings.
    - **`app/generate/`**: The page containing the form to generate new articles.
    - **`app/upgrade/`**: The page related to premium plans and monetization.
    - **`layout.tsx` & `page.tsx`**: The main application layout and homepage.

- **`components/`**: Reusable React components.
    - **`components/ui/`**: Core UI elements from Shadcn/UI (Button, Card, Input, etc.).
    - **`components/auth/`**: Components related to authentication (signin/signup forms).
    - **`components/admin/`**: Components for the admin dashboard.
    - **`generate-article-form.tsx`**: The form used on the `/generate` page.
    - **`article-content.tsx`**: Component for rendering the main content of a blog post.

- **`content/blog/`**: The location where the generated `.mdx` blog post files are stored.

- **`lib/`**: Core logic, helpers, and utilities.
    - **`lib/auth.ts`**: Authentication configuration and helper functions for NextAuth.
    - **`lib/db.ts` & `lib/schema.ts`**: Drizzle ORM setup, database connection, and table schemas.
    - **`lib/articles.ts` & `lib/mdx.ts`**: Utility functions for reading, parsing, and processing MDX blog content.

- **`drizzle/` & `scripts/`**:
    - **`drizzle/`**: Contains database migration files generated by Drizzle Kit.
    - **`scripts/`**: Node.js scripts for various development tasks like database seeding (`seed.ts`) and running migrations.

- **Configuration Files**:
    - **`next.config.ts`**: Next.js configuration.
    - **`drizzle.config.ts`**: Drizzle ORM configuration.
    - **`package.json`**: Project metadata, dependencies, and scripts.
    - **`README.md`**: Project setup and usage instructions.
